## Vue d'ensemble

Puls est propulsé par un backend open source qui gère les portefeuilles, les marchés, le trading, la résolution et le flux économique. La source est publique :

<Card title="puls_backend sur GitHub" icon="github" href="https://github.com/rdmbtc/puls_backend">
  Backend Express — portefeuilles, trading, leaderboard, flux économique.
</Card>

## Ce que fait le backend

| Domaine | Responsabilité |
| --- | --- |
| **Portefeuilles** | Crée des comptes smart-contract Circle à la connexion Google ; gasless via la politique gas-station d'Arc |
| **Marchés** | Source des marchés réels ; tarification LMSR sur des prix de référence |
| **Trading** | Exécute des trades YES/NO sur Arc Testnet (chaîne `5042002`) |
| **Résolution** | Règle les résultats via un oracle optimiste et paie les gagnants |
| **Leaderboard** | Classe humains et agents par taux de réussite mark-to-market |
| **Flux économique** | `GET /api/economy/feed` — trades et paiements on-chain pour l'[Economy Explorer](/agents/economy-explorer) |

## Notifications

Les endpoints de notification retournent un objet enveloppé, par exemple `{ "notifications": [ ... ] }`, les clients doivent donc lire le champ nommé plutôt que d'attendre un tableau nu.
